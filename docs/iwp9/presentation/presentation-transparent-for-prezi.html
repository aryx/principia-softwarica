How to use

Each frame below has a transparent background (the checkerboard shows through). To capture a frame for Prezi:

  1. Use your OS screenshot tool to select just the frame content
  2. On macOS: Cmd+Shift+4, drag to select
  3. The screenshot will have a transparent-looking background (actually the checkerboard)
  4. For truly transparent PNGs: right-click → Inspect → select the .frame div → right-click in Elements → "Capture node screenshot" (Chrome)

Chrome tip for true transparency: Open DevTools (F12), click the .frame div, then in Console type: copy(document.querySelector('#frame-name').outerHTML) — but for Prezi, node screenshots work best.

COLUMN 2: Fork Notes

Repository Structure

  • assemblers/5a/ not sys/src/cmd/5a
  • Each dir: .nw literate program + generated code
  • kernel/: concurrency/ memory/ processes/ …
  • mkconfig.pc (x86/QEMU) + mkconfig.pi (ARM/RPi)

Good strategy to find bugs: try to explain the code in a book.

COLUMN 2: Syncweb

Syncweb: Bidirectional LP

Classic Noweb: .nw ───► code (one way) Syncweb: .nw ◄──► code (both ways)

Marks + md5 checksums → detect which side changed
Automatic merge or conflict detection

Solves biggest LP complaint

DEMO

Live Demo: Modify, Rebuild, Boot

BEFORE: $ strace ls → open = syscall 6
CHANGE: $ vi lib_core/libc/9syscall/sys.h #define OPEN 6 → #define OPEN 42
REBUILD: $ time mk && mk install && mk kernel < 10 seconds
BOOT: $ qemu-system-i386 -kernel 9qemu -hda dosdisk.img $ strace ls → open = syscall 42

"Rebuilt the entire OS in under a minute. Try that with Linux."

That ls went through shell, libc, kernel, graphics, windowing. Every layer — explained in a Principia book.
