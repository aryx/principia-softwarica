TODO: candidates to write

[Kernel]
- Message passing vs shared memory: early OSes used messages (THE, RC 4000) ->
  shared memory + locks dominated (pthreads) -> message passing returns
  (Erlang, Go channels, actor model, Kubernetes pods communicating via gRPC)

- User-space file servers: Multics had user-space servers -> monolithic kernels
  put everything in-kernel -> FUSE (2005), 9P-over-virtio, io_uring bring
  file serving back to user space

[Linker Intro]
- Link-time optimization (LTO): early compilers did whole-program compilation
  (Fortran) -> separate compilation won (C, 1970s) -> LTO brings back
  cross-module optimization (GCC 4.5 2010, ThinLTO 2016)

[Compiler Intro]
- Interpreter comeback: interpreters (1950s LISP) -> compiled languages
  dominated for performance (1970s-1990s) -> JIT interpreters competitive
  (JVM HotSpot 1999, V8 2008, LuaJIT); also WASM as a new bytecode format

- S-expression / Lisp syntax: Lisp 1958 -> C-style syntax won -> Lisp ideas
  return as macros (Rust proc macros), homoiconicity (Clojure, Elixir),
  tree-structured editing (Paredit, structural editors)

- Garbage collection: McCarthy 1960 -> manual memory "real programmers" era ->
  GC dominates (Java 1995, Go, Python, JS); then Rust's borrow checker as
  a third path (compile-time GC). The full arc is manual -> GC -> Rust.

[Assembler Intro]
- Hardware description languages: assembly was "close to hardware" ->
  software abstracted away -> now hardware itself is programmable again
  (FPGAs, Verilog/VHDL, Chisel). Software people writing hardware.

[Machine]
- RISC vs CISC: RISC (Berkeley/Stanford 1980) -> CISC (x86) won market ->
  RISC returns: ARM dominates mobile, Apple M1 2020, RISC-V rising.
  Meanwhile x86 internally decodes to RISC micro-ops anyway.

- In-order execution: early CPUs all in-order -> out-of-order won (Pentium Pro
  1995) -> in-order returns for power efficiency (ARM Cortex-A53, Apple
  efficiency cores, embedded). Spectre/Meltdown made OoO's cost visible.

[Profiler]
- Instrumentation-based profiling: early profilers instrumented code ->
  sampling won (lower overhead) -> instrumentation returns as distributed
  tracing (OpenTelemetry spans are essentially instrumentation points)

- Hardware performance counters: existed since 1990s (Pentium) -> mostly
  ignored by developers -> perf (Linux 2009) and Intel VTune made them
  accessible; now eBPF makes them programmable

[Debugger]
- Record-and-replay debugging: early time-sharing had "snapshot" debugging ->
  printf/breakpoint dominated -> rr (Mozilla 2014), Pernosco bring
  deterministic replay back. Also: Undo, LiveRecorder.

- Reversible debugging: concept from 1960s (Balzer's EXDAMS) -> forward-only
  debuggers dominated -> rr + GDB reverse-step, UndoDB. "Step backwards"
  was a dream for 50 years.

[Shell Intro]
- Structured data in pipelines: early shells had typed/structured data ideas
  (not really, but Multics had structured records) -> text pipelines won
  (Unix) -> PowerShell objects (2006), Nushell tables (2019), jq for JSON.
  Plan 9's text-is-universal was a strength but structured data is returning.

[Utilities]
- Regular expressions: Kleene 1956, Thompson 1968 -> fell out of CS fashion
  (parser generators, formal grammars) -> regex is everywhere now (PCRE,
  every language's stdlib). Never truly died but had a "serious CS people
  don't use regex" period.

- Make/build systems: Make (Feldman 1976) -> complex build systems (Ant, Maven,
  Gradle, CMake) -> return to simplicity: ninja (2012) is basically "make
  done right", and Plan 9 mk was ahead of its time

[Editors]
- Modal editing: vi modes (1976) -> modeless won (Word, VS Code, most IDEs) ->
  Vim keybindings return everywhere (VS Code vim extension is the #1
  extension, Neovim renaissance, Helix, Kakoune). Millions of developers
  choosing modal editing in 2024.

- Terminal-based editors: vi/Emacs in terminal -> GUI editors won (1990s-2000s)
  -> terminal editors return: Neovim, Helix, Zed (hybrid). Even VS Code's
  terminal integration blurs the line.

[Windows]
- Tiling window managers: early window systems tiled (Xerox Cedar, Oberon) ->
  overlapping won (Mac, Windows) -> tiling returns: i3, sway, yabai, and
  Plan 9's rio was tiling-first all along. Also: VS Code/IDE splits are
  essentially tiling within an app.

- Server-side rendering (display servers): X11 network transparency (1984) ->
  local compositing won (Wayland, macOS Quartz) -> remote rendering
  returns as cloud desktops, VDI, browser-based IDEs (VS Code Server,
  GitHub Codespaces). Different technology, same "display is remote" idea.

[Lib_core]
- Cooperative multitasking at OS level: classic Mac OS cooperative (1984) ->
  preemptive won (NT, Unix) -> cooperative returns at app level (event
  loops, async runtimes). The OS is preemptive but apps internally are
  cooperative.

- Green threads / M:N threading: early systems used M:N (Solaris LWP) ->
  1:1 kernel threads won (Linux NPTL 2003) -> M:N returns: Go goroutines,
  Java virtual threads (Project Loom 2023), Rust tokio. Plan 9's libthread
  was M:N from the start.

[Lib_gui]
- Immediate-mode GUI: early graphics were immediate (draw, forget) ->
  retained mode won (scene graphs, DOM, widget trees) -> immediate mode
  returns: Dear ImGui (2014), game UIs, Egui (Rust). The React virtual
  DOM is conceptually immediate-mode with a retained-mode reconciler.
