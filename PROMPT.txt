Principia softwarica is essentially a set of books derived from literate
program files (.nw) that cover the set of programs essential for a programmer.

Many of those books are almost done: Make.nw, Assembler.nw, Linker.nw,
or have the code spread in chapters and with an intro: Graphics.nw, Windows.nw,
or just the code spread: Kernel.nw, Shell.nw, etc.

The goal is to finish those books, to add explanations before the
noweb chunk code in those .nw files.

You should read the many .nw files in this project, especially the one
that are almost complete, to infer my style of presentation, what I like
to insist on. For example I don't explain too much trivial error code management
or just introduce it once. I also tend to use those %toc: and %trans:
(and many more) special "tags" to classify the kind of explanation I write.
update: see also now docs/latex/Tags.tex

I also tend to introduce figures using ASCII art (memory layout, sequence
diagrams, control flow diagrams, and many more); feel free to do the same
if it helps understanding the code sometimes.

Sometimes the code should be further splitted to help make the explanation
manageable, but for now let's keep the chunks as-is and just add some
TODO notes before if you think it should be further split.

What I would like from you is to complete those books, one by one,
section by section, interactively.

We can start with XXX.nw that has just an (incomplete intro) and
we can start from the Core data structure chapter and then go on with
the rest one by one.

Please in all the modifications you've done, add a %claude: before so I know
you wrote that part and leave back the original comment (lines that start with '%') that I wrote
about the stuff I wanted to say, so I can check if your explanation satisfy
my TODO comments; same with my \l, \t todo annotations (\n is for notes and
is internal only so does not need comments.

Also Intro.nw has lots of comments for things I wanted to explain. Again would be  
good if you add explanations that "implement" those TODO comments
(but leave the comments).

Again you can look at the Intro.nw for the other programs (Intro.nw of
assembler/, of builder/, ...) as a model.

No need to add lots of explanations for code in the appendix.

And also try to explain things that are subtle and not obvious from the code,
like I did in the other books. Do not "over-explain". I assume
the people who will read those books are good programmers who can understand
most of the code in the noweb chunks, but need just a little help to
make the flow to comprend the code easier, especially the global picture.
